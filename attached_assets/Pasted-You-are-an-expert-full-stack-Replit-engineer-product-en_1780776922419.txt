PRODUCT DIRECTION

Cerevia is not a migraine tracker.

Cerevia is a personalised migraine trigger management platform.

The core product loop is:

Track → Discover → Manage → Prevent

The app helps migraine sufferers:

1. Track potential migraine triggers
2. Log migraine events
3. Discover their personal trigger patterns
4. Receive personalised trigger management actions
5. Reduce exposure to triggers
6. Reduce migraine frequency and daily disruption
7. Generate clinician-ready summaries when needed

The primary value proposition is:

Cerevia helps migraine sufferers identify and manage their personal triggers to reduce migraine frequency and improve quality of life.

The app must move away from:

* Generic symptom tracking
* Generic health tracking
* Broad wellness tracking
* Simple migraine diary behaviour
* Passive charts with no action

The app must move toward:

* Trigger intelligence
* Trigger management
* Behaviour change
* Personalised prevention
* Quality-of-life improvement

CORE PRODUCT PRINCIPLES

1. Migraines are the outcome metric.
2. Triggers are the main thing being tracked.
3. Insights must lead to action.
4. The app should help the user reduce exposure to triggers.
5. The app should become more personalised over time.
6. The product should feel clinically responsible and safe.
7. The app must not diagnose, prescribe, recommend medication changes or replace clinical judgement.
8. Use safe language such as:

   * “appears associated with”
   * “may be worth discussing”
   * “reported”
   * “based on your logged data”
   * “personal pattern”
   * “trigger exposure”
   * “management focus”
9. Avoid unsafe language such as:

   * “this causes your migraine”
   * “you should stop medication”
   * “diagnosis”
   * “treatment recommendation”
   * “medical advice”

AI / INSIGHT ENGINE

For Replit, implement AI in a practical way.

First use mock/static insight generation.
Then add OpenAI API only if environment variables are available.

Required AI functions:

1. generateTriggerProfile(userData)
   Returns ranked triggers and confidence scores.

2. generateTriggerCoach(triggerProfile)
   Returns personalised trigger management actions.

3. generateGPSummary(userData)
   Returns clinician-ready summary.

4. answerAskCerevia(question, userData)
   Returns grounded answer based on logs.

5. structureVoiceNote(transcript)
   Optional. Turns free text into trigger or migraine event data.

MVP can use deterministic mock logic:

* If sleep < 6h appears before multiple migraines, increase sleep trigger confidence.
* If stress > 7 appears before multiple migraines, increase stress trigger confidence.
* If meals_skipped is true before multiple migraines, increase skipped meals trigger confidence.
* If caffeine is inconsistent before migraines, flag caffeine consistency.
* If migraine frequency drops while trigger exposure improves, show progress.

Do not overcomplicate with ML.

Use clear rule-based calculations for the hackathon.
