
ASK CEREVIA

If implemented, Ask Cerevia should answer questions about the user’s own logged data.

Example questions:

* What are my strongest triggers?
* What changed before my migraines got worse?
* Does sleep seem linked to my migraines?
* Which triggers should I focus on this week?
* Has my medication helped reduce duration?
* What should I discuss with my GP?

Answers must be grounded in logged data and avoid diagnosis.

Example:
“Based on your logs, sleep appears to be your strongest associated trigger. Severe migraines were more common after nights with less than 6 hours sleep. This may be worth discussing with your GP if migraines are increasing.”

REMINDERS SCREEN

The current Reminders section should become trigger-management reminders.

Not generic reminders.

Types:

* Sleep wind-down reminder
* Hydration reminder
* Meal consistency reminder
* Screen break reminder
* Stress reset reminder
* Trigger check-in reminder
* Medication reminder if user already takes migraine medication

Each reminder should be linked to a trigger management goal.

Example:
“Screen break reminder is active because long screen sessions appear associated with your migraine reports.”

AI / INSIGHT ENGINE

For Replit, implement AI in a practical way.

First use mock/static insight generation.
Then add OpenAI API only if environment variables are available.

Required AI functions:

1. generateTriggerProfile(userData)
   Returns ranked triggers and confidence scores.

2. generateTriggerCoach(triggerProfile)
   Returns personalised trigger management actions.

3. generateGPSummary(userData)
   Returns clinician-ready summary.

4. answerAskCerevia(question, userData)
   Returns grounded answer based on logs.

5. structureVoiceNote(transcript)
   Optional. Turns free text into trigger or migraine event data.

MVP can use deterministic mock logic:

* If sleep < 6h appears before multiple migraines, increase sleep trigger confidence.
* If stress > 7 appears before multiple migraines, increase stress trigger confidence.
* If meals_skipped is true before multiple migraines, increase skipped meals trigger confidence.
* If caffeine is inconsistent before migraines, flag caffeine consistency.
* If migraine frequency drops while trigger exposure improves, show progress.

Do not overcomplicate with ML.

Use clear rule-based calculations for the hackathon.

FINAL PRODUCT POSITIONING

Cerevia is a personalised migraine trigger management platform.

It helps users track potential triggers, discover their individual migraine patterns and take personalised actions to reduce trigger exposure.

The product’s success is measured by:

* Fewer migraine days
* Lower severity
* Reduced life disruption
* Better trigger control
* Better GP conversations

Do not build a generic tracker.

Build a product that helps migraine sufferers answer:

“What is causing my migraines, what can I control, and what should I focus on today?”
